    Introduction

    Mock-SMS API is a mock sms management system API that allows easy operations for messages. Contacts can create, retrieve and delete messages. Contacts can also see their sent and received messages

    Development

    The application was developed with NodeJs and Express for routing. Postgres is the database used and sequelize is the ORM used.

    Installation

    Follow the steps below to setup a local development environment. First ensure you have Postgresql installed, and a version of Node.js equal or greater than v6.8.0.

    1. Clone the repository from a terminal git clone https://github.com/joel-ace/mock-sms-api.git.
    2. Change into the mock-sms-api directory cd mock-sms-api
    3. Rename .env_sample to .env and add the required DATABASE URL.
    4. Install project dependencies npm install
    5. Start the express development server npm run dev.

    API Summary

    Contacts

    EndPoint Functionality
    POST /api/v1/contacts/ Creates a new contact.
    GET /api/v1/contacts/ Gets all contacts.
    GET /api/v1/contacts/:id Gets contact using it's id.
    DELETE /api/v1/contacts/:id Deletes contact
    GET /api/v1/contacts/:id/sms/sent Gets all messages sent by a contact
    GET /api/v1/contacts/:id/sms/received Gets all messages received by a contact

    Messages

    EndPoint Functionality
    POST /api/v1/messages/ Creates a new message.
    GET /api/v1/messages/:id Gets a message using it's id.
    DELETE /messages/:id Deletes message

    Errors

    If you are new to HTTP status codes, you can use this as a reference to know what the returned error status codes used in Mock-SMS API means.

    Mock-SMS API uses the following code error codes:

    Error Code Meaning
    400 Bad Request -- You're sending a request with bad/not properly formatted parameters
    404 Not Found -- What you requested for could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
